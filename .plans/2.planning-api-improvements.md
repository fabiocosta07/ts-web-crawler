# Plan: ts-web-crawler API improvements

**Date:** 2026-04-18
**Status:** in-progress
**Scope:** `/home/fabio/.distrobox-homes/dev/repos/easy-workout-monorepo/ts-web-crawler`

## Context

The API has a clean, small surface (auth + crawl-jobs + BFS engine) but three
parallel review passes surfaced concrete improvements across security,
efficiency, and DRY. No pending diff ‚Äî this plan reviews the checked-in code
as of commit `f965c01`. The aim is a tight, surgical cleanup that fixes real
defects and duplication without redesigning the project.

Priorities below are ordered by impact (correctness/security first, then
performance, then code quality). Each item lists the target file(s) and the
specific change.

User-confirmed scope: **HIGH + MEDIUM** (H1‚ÄìH5, P1‚ÄìP4, Q1‚ÄìQ6). LOW items
optional. `robots-parser` to be removed, not wired up.

## Goals

- Remove a latent prod-secret-leak footgun (JWT default).
- Stop a hot-path N-serial-fetch bottleneck in the crawler.
- Tighten request validation so bad input can't reach Prisma or the engine.
- Eliminate duplicated validation / ownership / mapping boilerplate.
- Respect the site being crawled (bounded concurrency, page cap).

## Non-goals

- No change to the public API shape of existing endpoints.
- No new routes, no repository layer, no DI refactor.
- No new dependencies if an existing one or a trivial inline version will do.
  (Exception: `express-rate-limit` for auth endpoints.)
- No CI / observability stack changes ‚Äî stick to in-process fixes.

## Recommended changes

### HIGH ‚Äî correctness & security

**H1. Centralize `JWT_SECRET`, fail closed in production.**
Duplicated in `src/routes/auth.ts:7` and `src/middleware/auth.ts:4` with the
same dangerous default `'change-me-in-production'`. Create
`src/lib/config.ts` exporting a single `JWT_SECRET` constant that throws at
module load if `process.env.JWT_SECRET` is unset **and** `NODE_ENV ===
'production'`. Import from both callers.

**H2. Validate query params with Zod on the list endpoint.**
`src/routes/crawl-jobs.ts:11-12`: `Number(req.query.cursor ?? 0)` yields
`NaN` for bad input and is silently passed to Prisma as `skip`. Add
`z.object({ cursor: z.coerce.number().int().min(0).default(0), limit:
z.coerce.number().int().min(1).max(100).default(20) })` and `safeParse`
`req.query`.

**H3. Raise password strength & add rate limiting to auth.**
- `src/routes/auth.ts:11`: `z.string().min(1)` ‚Üí `z.string().min(8)`.
- Add `express-rate-limit` scoped to `authRouter` ‚Äî 10 req/min per IP for
  `/signup` and `/login`.

**H4. Remove the `req.userId!` non-null assertion.**
`src/routes/crawl-jobs.ts:97`. Change `AuthRequest.userId` to required and
only set it after verification ‚Äî guarantee by construction.

**H5. Robust failure persistence.**
`src/crawler/engine.ts:165-172` already sets `FAILED` on throw and
`src/routes/crawl-jobs.ts:102-104` re-catches. Wrap the inner FAILED update
in its own try/catch so a DB hiccup there is logged, not masked.

### HIGH ‚Äî performance

**P1. Parallelise the BFS level in the crawler.**
`src/crawler/engine.ts:118-155`: currently serial per-URL awaits. Replace
with a bounded-concurrency runner (`CONCURRENCY = 5`) per level. No new
dep ‚Äî inline worker-pool or chunked `Promise.all`.

**P2. Paginate results on `GET /api/crawl-jobs/:id`.**
`src/routes/crawl-jobs.ts:41-76`: split into job fetch + paginated
`crawlResult.findMany` driven by the schema from H2. Include
`resultCount` from `_count`.

**P3. Per-job resource caps.**
`src/crawler/engine.ts`: `MAX_PAGES_PER_JOB` (env, default 500). Cap
per-fetch response at ~2 MB by reading with a stream.

**P4. Index `CrawlJob.userId`.**
`prisma/schema.prisma`: add `@@index([userId])` + migration
`add-crawl-job-user-index`.

### MEDIUM ‚Äî code quality & reuse

**Q1. `validateBody(schema)` middleware.**
`src/middleware/validate.ts` ‚Äî single source for `safeParse` + 400. Consume
from auth.ts √ó 2 and crawl-jobs.ts √ó 1.

**Q2. `CrawlStatus` enum from Prisma.**
`src/crawler/engine.ts:107, 163, 169`: replace string literals with enum.

**Q3. Collapse ownership checks.**
`src/routes/crawl-jobs.ts`: use `findFirst({where: {id, userId}})` for GET
and `deleteMany({where: {id, userId}})` for DELETE.

**Q4. Remove `robots-parser`.**
Unused in code; drop from `package.json`.

**Q5. Delete WHAT-comments in `engine.ts`.**
Lines 19-21, 38-39, 100-103.

**Q6. Graceful shutdown.**
`src/index.ts`: SIGTERM handler that `server.close()` + `prisma.$disconnect()`.
